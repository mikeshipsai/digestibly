# Монетизация Digestibly

Документ описывает варианты монетизации персонального Telegram-дайджест бота. Продукт сейчас — self-hosted проект для одного владельца с Telethon userbot.

---

## Что продаётся на самом деле

Не «бот в Telegram», а **персональный фильтр информационного шума**:

- автоматический сбор из подписок;
- тематическая группировка;
- LLM-саммари и top-N;
- доставка в удобное время.

**Ценность для пользователя:** экономия 30–60 минут в день на просмотре каналов.

---

## Ключевое ограничение

Текущая архитектура завязана на **Telethon userbot** — он читает **все broadcast-каналы личного аккаунта** пользователя. Это влияет на модель монетизации:

- для SaaS нужна **отдельная сессия на каждого пользователя**;
- хранение `.session` — высокий уровень доверия и риски безопасности;
- массовый userbot — **серая зона** относительно правил Telegram (теоретический риск ограничений аккаунта).

---

## Три модели монетизации

### 1. Open-source + «сделай сам»

**Суть:** репозиторий остаётся бесплатным; монетизация через помощь в развёртывании и сопровождение.

| Вариант | Цена (ориентир) | Что продаётся |
|---------|-----------------|---------------|
| Подробный setup-guide + видео | 990–2 990 ₽ разово | «Подключи за один вечер без боли» |
| Managed VPS setup | 3 000–5 000 ₽ разово | Развёртывание на сервере клиента |
| «Digestibly Pro» Docker + env wizard | 499 ₽/мес | Готовый образ + обновления |

**Плюсы:**

- не нужен multi-tenant;
- нет хранения чужих Telegram-сессий;
- меньше юридических и технических рисков.

**Минусы:**

- аудитория в основном техническая;
- невысокий чек;
- поддержка («почему не работает Groq»).

**Целевая аудитория:** разработчики, PM, аналитики с 50+ каналами — те, кто уже платит за VPS и API.

---

### 2. SaaS по подподписке

**Суть:** пользователь не деплоит сам — инстанс хостится у вас (отдельный worker или изолированный контейнер на пользователя).

#### Экономика на одного пользователя (оценка)

| Статья | ₽/мес |
|--------|-------|
| VPS (доля на user при 20–50 users на ноде) | 50–150 |
| LLM (Gemini free → Groq, 100–300 постов/день) | 0–300 |
| Поддержка (амортизация) | 50–100 |
| **Себестоимость** | **~100–400** |

#### Тарифная сетка (рекомендация для RU)

| Тариф | Цена | Лимиты |
|-------|------|--------|
| **Free** | 0 | до 30 каналов, 1 digest/день, общая очередь batch |
| **Personal** | **399 ₽/мес** | до 100 каналов, приоритет batch |
| **Pro** | **699 ₽/мес** | до 200 каналов, weekly digest, персонализация 👍/👎 |
| **Pro+** | **1 290 ₽/мес** | расширенные лимиты, опционально свой Groq key |

**299 ₽/мес** — возможен как **промо** (первый месяц или акция), но постоянная цена **399–699 ₽** реалистичнее с учётом LLM и поддержки.

#### Что нужно доработать в продукте

- multi-user: `owner_chat_id` → таблица пользователей;
- отдельная Telethon-сессия на пользователя;
- billing: ЮKassa, Telegram Stars или Robokassa;
- onboarding без SSH: API ID → код входа → готово;
- изоляция данных (SQLite per user или `tenant_id` везде);
- лимиты и fair use.

#### Риски

- Telegram ToS и userbot при масштабе;
- утечка `.session` = полный доступ к аккаунту пользователя;
- free tier Groq (digest 30–60 мин) неприемлем для SaaS без платных API.

---

### 3. B2B / white-label

**Суть:** не личный дайджест из всех подписок, а **дайджест для команды** из фиксированного списка каналов.

**Примеры:**

- маркетинг: дайджест конкурентов и индустрии;
- VC/фонд: новости по портфелю;
- медиа: internal morning brief.

**Цена:** 5 000–30 000 ₽/мес за команду (5–20 seats).

**Техника:** публичные каналы через Bot API, shared userbot или RSS — проще compliance, чем «все подписки сотрудника».

Это **отдельный продукт**, но монетизация проще и чек выше.

## Рекомендуемая стратегия (поэтапно)

```
Сейчас: self-host OSS
    ↓
Фаза 1: Paid setup kit (990–1 990 ₽)
    ↓
Фаза 2: Hosted Personal (399 ₽/мес) — если есть спрос
    ↓
Фаза 3: Team / B2B
```

### Фаза 1 — быстрые деньги без переписывания архитектуры

1. **«Digestibly Setup»** — платный Notion/PDF + видео:
   - BotFather, my.telegram.org, `.env`, VPS, systemd;
   - типичные ошибки (session conflict, Groq 429).
2. Telegram-канал с кейсами + ссылка «купить setup» (Boosty, Gumroad, ЮKassa).
3. Donation / GitHub Sponsors для OSS-аудитории.

**Чек:** 990–1 990 ₽ разово.  
**Цель:** 10–30 продаж — проверка спроса без SaaS.

### Фаза 2 — подписка (только при спросе после Фазы 1)

**«Digestibly Cloud»:**

- пользователь: `/start` → flow Telethon (код в Telegram) → подписка;
- инфраструктура: VPS на N пользователей, worker per session;
- **399 ₽/мес**, trial 7 дней.

Критично для conversion: прозрачность пайплайна и sample digest при onboarding (см. `TODO.md`).

### Фаза 3 — корпоративные запросы

Тариф «Team digest» без доступа к личным подпискам сотрудника.

## Сравнение моделей

| Модель | Сложность | Деньги | Риск |
|--------|-----------|--------|------|
| Инструкция + setup | Низкая | Низкие, быстрые | Минимальный |
| SaaS 399 ₽/мес | Высокая | Средние при 100+ users | Session, ToS, LLM |
| B2B team digest | Средняя | Высокие | Другой продукт сбора данных |

---

## Вывод

- **299 ₽/мес** — возможен как промо; постоянная цена **399–699 ₽** реалистичнее.
- **Самый разумный первый шаг:** платный setup-kit + landing + waitlist на Cloud. При 50+ заявках в waitlist — invest в multi-tenant.
- Модель «создай своего бота и подключи проект» хорошо ложится на **Фазу 1**, не требует переписывания кода.
